TDD Cycle


1. Write a (failing) test

2. Write the code

3. Run tests

4. Refactor (or fix the code and go back to 3)

TDD Benefits


1. Better design

2. Shorter development time (source)

3. Fewer bugs (source)

4. It's professional (solutions come after a problem is formulated)
